Git Tag adalah penanda yang digunakan untuk menandai sebuah commit spesifik dalam riwayat proyek. Ini sangat berguna untuk menandai poin-poin penting, seperti rilis versi (contoh: v1.0, v2.0.1).

Ada dua jenis tag utama: annotated (disarankan) dan lightweight.

Cara Menggunakan Git Tag

Membuat Annotated Tag (Disarankan)

Jenis ini menyimpan informasi tambahan seperti nama penanda, email, tanggal, dan pesan. Ini adalah praktik terbaik untuk rilis resmi.

git tag -a v1.0 -m "Pesan rilis, contoh: Versi 1.0 stabil"

Membuat Lightweight Tag

Jenis ini hanyalah sebuah penunjuk ke sebuah commit, tanpa informasi tambahan. Cocok untuk penanda pribadi atau sementara.

git tag v1.0-beta

Melihat Semua Tag

Untuk menampilkan daftar semua tag yang ada di repositori lokal.

git tag

Melihat Detail Tag

Menampilkan informasi lengkap dari sebuah annotated tag, termasuk pembuat dan pesannya.

git show v1.0

Mendorong (Push) Tag ke Remote

Secara default, `git push` tidak mengirim tag. Anda harus mendorongnya secara eksplisit.

# Mendorong satu tag spesifik git push origin v1.0 # Mendorong semua tag yang ada di lokal git push origin --tags

Menghapus Tag

Menghapus tag dari repositori lokal dan remote jika diperlukan.

# Menghapus tag lokal git tag -d v1.0 # Menghapus tag di remote (setelah dihapus di lokal) git push origin --delete v1.0

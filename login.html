Menu Perintah Git Cafe!

Selamat datang di area ngoding! Ini dia perintah-perintah Git yang sering kamu pakai:

Git Init

Membuat repositori Git baru di folder saat ini.

git init

Git Add

Menambahkan perubahan file ke area persiapan (staging area) untuk di-commit.

git add <nama-file> git add . (untuk semua perubahan)

Git Commit

Menyimpan perubahan yang sudah di-add ke riwayat repositori dengan pesan.

git commit -m "Pesan commitmu di sini"

Git Status

Melihat status repositori: file apa yang sudah diubah, yang belum di-add, dll.

git status

Git Log

Melihat riwayat commit repositori.

git log

Git Branch

Melihat daftar branch yang ada atau membuat branch baru.

git branch (melihat) git branch <nama-branch-baru> (membuat)

Git Checkout

Berpindah antara branch atau versi commit tertentu.

git checkout <nama-branch> git checkout <id-commit>

Git Merge

Menggabungkan perubahan dari satu branch ke branch lain.

git merge <nama-branch-sumber>

Git Pull

Mengambil perubahan terbaru dari repositori remote (online) dan menggabungkannya ke repositori lokal.

git pull origin <nama-branch>

Git Push

Mengirim perubahan dari repositori lokal ke repositori remote (online).

git push origin <nama-branch>

Git Clone

Mengambil salinan lengkap repositori dari remote ke komputer lokalmu.

git clone <url-repositori>

Git Remote

Mengelola koneksi ke repositori remote.

git remote add origin <url-repositori> git remote -v

Git Reset

Mengembalikan status repositori ke commit tertentu (hati-hati!).

git reset --hard <id-commit> git reset <nama-file> (membatalkan add)

Git Revert

Membuat commit baru yang membatalkan perubahan dari commit sebelumnya (lebih aman dari reset).

git revert <id-commit>

Git Config

Mengatur konfigurasi Git lokal atau global.

git config --global user.name "Nama Kamu" git config --global user.email "email@contoh.com"

Git Tag

Memberi tanda (tag) pada commit tertentu, biasanya untuk versi rilis.

git tag -a v1.0 -m "Versi 1.0" git push origin --tags

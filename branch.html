Dalam Git, branch (cabang) adalah sebuah penunjuk yang bisa berpindah-pindah ke salah satu dari serangkaian *commit*. Anggap saja seperti sebuah alur kerja atau lini pengembangan yang terisolasi. *Branch* utama dalam sebuah repositori biasanya bernama main atau master.

Dengan menggunakan *branch*, Anda dan tim Anda dapat mengerjakan berbagai fitur atau perbaikan secara bersamaan dalam proyek yang sama tanpa mengganggu alur kerja utama. Setelah pekerjaan di sebuah *branch* selesai, perubahan tersebut dapat digabungkan kembali ke *branch* utama.

Perintah-Perintah Dasar Branch

Berikut adalah beberapa perintah yang paling sering digunakan untuk mengelola *branch* di Git.

Membuat Branch Baru

Perintah ini akan membuat sebuah *branch* baru, namun Anda masih berada di *branch* saat ini.

git branch [nama-branch-baru]

Melihat Semua Branch

Untuk melihat daftar semua *branch* yang ada di repositori lokal Anda. *Branch* yang sedang aktif akan ditandai dengan tanda bintang (*).

git branch

Berpindah Branch

Untuk beralih dari *branch* saat ini ke *branch* lain yang sudah ada.

git checkout [nama-branch-tujuan]

Membuat dan Langsung Berpindah Branch

Ini adalah cara cepat untuk membuat *branch* baru dan langsung beralih ke *branch* tersebut dalam satu perintah.

git checkout -b [nama-branch-baru]

Menggabungkan Branch (Merge)

Untuk menggabungkan perubahan dari *branch* lain ke *branch* Anda saat ini. Pastikan Anda sudah berada di *branch* tujuan sebelum menjalankan perintah ini.

git merge [nama-branch-sumber]

Menghapus Branch

Setelah *branch* selesai digunakan dan perubahannya sudah digabungkan, Anda bisa menghapusnya.

git branch -d [nama-branch-yang-akan-dihapus]

Gunakan -D (huruf besar) untuk memaksa penghapusan jika *branch* tersebut memiliki perubahan yang belum digabungkan.

git branch -D [nama-branch]

Mengganti Nama Branch

Untuk mengubah nama branch lokal saat ini.

git branch -m [nama-baru]
